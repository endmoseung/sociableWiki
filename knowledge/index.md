# Topics

Every doc is one **concept** — a single sharp claim or decision rule. The concept id is the path you pass to `read_doc` (e.g. `ai-native/fan-out-scope-gate`).

## ai-native

Working with coding agents, harness design, subagent orchestration, LLM evaluation, and context management.

- **[/clear vs /compact vs autocompact — decide at the task seam, not at a token threshold](ai-native/clear-vs-compact-vs-autocompact.md)** — The context write-action you pick is chosen at the boundary between sub-tasks, not when the fill gauge crosses a percentage.
- **[A Dynamic Workflow Is the Wrong Default — the Anti-Trigger Is Its Own Skill](ai-native/dynamic-workflow-anti-trigger.md)** — Knowing when NOT to author a dynamic workflow is a distinct skill; default to the rung below one and let the task force you up.
- **[Agent framework fatigue: when to skip the framework and use raw APIs or vendor SDKs](ai-native/agent-framework-fatigue.md)** — Reach for LangGraph/CrewAI only when the problem is graph-shaped, cross-session, or genuinely multi-agent — otherwise a raw API or vendor SDK ships faster with less lock-in.
- **[Agent Memory Architecture — Tiers, Control, and the Debuggability Gap](ai-native/agent-memory-architecture.md)** — Agent memory is a managed lifecycle across active/episodic/semantic tiers, and its central unsolved problem is attributing an error across retrieval, write-path, compression, and reasoning.
- **[Authoring a Claude Code plugin: the checklist local testing misses (paths · versions · triggers)](ai-native/plugin-authoring-checklist.md)** — A plugin that works in a local .claude/ dir can silently break once it ships and gets installed; three axes — paths, versions, triggers — catch almost all of it.
- **[Batch PR Review with a Parallel Subagent Pipeline](ai-native/batch-pr-review-pipeline.md)** — Review many open PRs at once with one subagent per PR, then map inline comments using the files-API patch — not the pr-diff line numbers — or GitHub rejects them.
- **[Best-of-N coding agents — spend tokens to buy variance, then discard](ai-native/best-of-n-coding-agents.md)** — Run the same task in N isolated worktrees, keep exactly one output, and throw the other N−1 away — you pay N× tokens to buy variance, not to merge.
- **[Cache-stable harness design — \"don't break the prefix\" as a cross-vendor discipline](ai-native/cache-stable-harness-design.md)** — In a long agent loop the prompt cache is a structural property of how you rebuild the prompt every turn, not a flag — you preserve hits by construction, and small harness decisions silently bust them.
- **[Claude Code /rewind undoes by which tool wrote the change, not by what the agent did](ai-native/rewind-coverage-gap.md)** — /rewind's undo coverage is keyed to the editing tool that produced a change, so bash and external side-effects — where most irreversible damage lives — fall outside the safety net.
- **[Coding-agent cost economics — token budgeting and model×effort routing](ai-native/agent-cost-economics-routing.md)** — Most agent spend is a routing problem, not a reasoning problem; in 2026 routing became two axes — which model × how much effort — and subagents buy clean context at ~7× the token volume.
- **[Coding-agent primitive selection map — CLAUDE.md vs Skill vs Subagent vs Hook vs MCP](ai-native/agent-primitive-selection-map.md)** — A per-behavior routing table for which Claude Code primitive to encode a reusable behavior in, picked in priority order.
- **[Coding-agent reward hacking: measure the held-out gap, not the pass rate](ai-native/reward-hacking-held-out-gap.md)** — A visible-suite pass no longer proves anything; the gap between visible and held-out tests is the real reward-hacking signal, and it grows with task size.
- **[Commit attribution is a write to shared history, not a credit line](ai-native/commit-attribution-governance.md)** — The trailer choice (Co-Authored-By vs Assisted-by vs Generated-by) is a per-repo governance decision graded by who reads git history — and the real hazard is the default-on write, not the trailer itself.
- **[A review finding is a hypothesis until a different reviewer fails to disprove it](ai-native/independent-verification-of-review-findings.md)** — AI-generated review comments should pass an independent disconfirmation step before publication because plausible findings often collapse under surrounding context.
- **[Enforcement stacking is a floor, not a ceiling — long-horizon drift needs its own detector](ai-native/enforcement-floor-vs-drift-detector.md)** — A full hook + memory + skill stack raises the per-turn floor but does not cap multi-day architectural drift, which needs a separate periodic conformance check.
- **[Fan-out width has two ceilings — cost is linear, synthesis fidelity is the one that sets 3–5](ai-native/fan-out-width-ceiling.md)** — The everyday 'spawn 3–5 subagents' rule is set by the lead's ability to reconcile divergent returns, not by token cost — and it only binds when the returns must be merged into one answer.
- **[Fork vs fresh subagent: the cache-reuse gate after isolation pays off](ai-native/fork-vs-fresh-subagent.md)** — Once you've decided to spawn, forking the parent vs opening a fresh subagent is a prompt-cache economics decision, not a capability one.
- **[Sandboxing, not permissions, is what makes a coding agent autonomous](ai-native/sandboxing-enables-autonomy.md)** — An OS-level sandbox — not a pile of permission prompts — is what lets a coding agent run unattended, because it contains the blast radius instead of asking you to catch every risky action.
- **[Review large changes by dependency cluster, not by file count](ai-native/dependency-clustered-code-review.md)** — A large change stays understandable when files are grouped by imports, calls, shared contracts, and data flow so each reviewer sees a complete behavior slice.
- **[Silent Default Changes Are Breaking Behavior Changes — Three Postmortem Lessons](ai-native/silent-default-change-postmortem.md)** — When a model platform quietly shifts an intelligence-affecting default, the caller sees degraded output with no error and no code change on their end — so you have to detect it from usage telemetry, not from a failure signal.
- **[The eval sandbox is an attack surface: shared state between agent and verifier](ai-native/eval-sandbox-attack-surface.md)** — If the coding agent and the verifier share mutable state inside one container, the agent can pre-arm the grader to lie — so the leaderboard number is fiction.
- **[The fan-out scope gate — freeze a filter into a manifest before spawning a batch](ai-native/fan-out-scope-gate.md)** — Before spawning N workers, resolve the filter to an exact list, get the human to confirm it, and freeze it into a claim manifest — workers never see a live filter.
- **[The Harness Is the Eval Unit](ai-native/harness-is-the-eval-unit.md)** — When you evaluate an LLM agent, the scaffold shifts benchmark scores as much as the model does — so hold the harness constant, and calibrate any LLM-as-judge with a cross-family model before trusting it.
- **[The Tool Receipt Pattern and Why Rubric Quality Is the Eval Bottleneck](ai-native/agent-eval-tool-receipt-pattern.md)** — In agent evaluation, the limiting factor is rubric design, not judge model strength — and the sharpest hallucination catcher is verifying that claimed tool calls actually ran.
